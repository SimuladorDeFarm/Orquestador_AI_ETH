## 5. Vectores de Ataque Sugeridos para la Siguiente Fase

1. **Fuerza bruta en FTP (puerto 21):** Dado que el acceso anónimo no permitió listar archivos, se recomienda intentar fuerza bruta de credenciales contra el usuario `anonymous` o usuarios comunes (admin, ftp, etc.) para obtener acceso completo al servidor FTP. Especialmente relevante si vsftpd tiene vulnerabilidades conocidas.

2. **Enumeración del directorio `/javascript/`:** El acceso denegado (403) es inusual para un directorio de scripts JS. Se recomienda:
   - Realizar fuzzing con wordlist específica para archivos JavaScript comunes (`script.js`, `main.js`, `api.js`, etc.)
   - Probar extensiones adicionales (`.php`, `.txt`, `.bak`, etc.)
   - Verificar si incluye archivos con nombres basados en los personajes de Cowboy Bebop

3. **Explotación de vulnerabilidades en Apache 2.4.41:** 
   - Verificar si la versión es vulnerable a path traversal o LFI (ej: CVE-2021-41773, CVE-2021-42013)
   - Intentar acceder a `../` o codificación URL para el directorio `/javascript/`

4. **Fuzzing más agresivo en HTTP:**
   - Probar rutas con palabras relacionadas a Cowboy Bebop (`spike`, `jet`, `ed`, `faye`, `bebop`, `swordfish`)
   - Buscar archivos de respaldo (`.bak`, `.old`) o archivos de configuración

5. **Análisis de la imagen `/images/crew.jpg`:** 
   - Descargar `crew.jpg` y verificar si contiene datos ocultos mediante esteganografía (herramientas como `steghide`, `binwalk`, `strings`)
   - Revisar metadatos EXIF para posibles pistas

6. **Verificar SSH (puerto 22):** 
   - Enumerar usuarios válidos mediante fuerza bruta o timed-based enumeration
